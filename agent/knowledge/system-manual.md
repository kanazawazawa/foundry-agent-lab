# TravelExpense システム 操作マニュアル

## 1. システム概要

TravelExpense は、出張申請・承認・精算を一元管理する Web アプリケーションです。

- **URL**: https://travelexpense.example.com
- **対応ブラウザ**: Microsoft Edge、Google Chrome（最新版）
- **ログイン**: 社内 SSO（Microsoft Entra ID）で自動認証

## 2. 出張申請の手順

### 2.1 新規出張申請

1. トップ画面の **[新規申請]** ボタンをクリック
2. 申請種別で **「出張申請」** を選択
3. 以下の項目を入力する
   - **出張先**: 都道府県および市区町村（海外の場合は国名・都市名）
   - **出張目的**: プルダウンから選択（顧客訪問 / 社内会議 / 研修 / 展示会 / その他）
   - **出張期間**: カレンダーから開始日・終了日を選択
   - **同行者**: 同行者がいる場合は社員番号または氏名で検索して追加
   - **交通手段**: チェックボックスで選択（鉄道 / 新幹線 / 航空機 / バス / タクシー / レンタカー / 自家用車）
   - **概算金額**: 交通費・宿泊費・日当の概算を入力
4. **[確認画面へ]** ボタンをクリック
5. 入力内容を確認し、**[申請する]** ボタンをクリック
6. 承認者に自動で通知メールが送信される

### 2.2 下書き保存

- 申請画面の **[下書き保存]** ボタンをクリックすると、入力途中の内容を保存できる
- 下書きは **[マイページ] → [下書き一覧]** から再開できる
- 下書きの保存期限は **30日間**。期限を過ぎると自動削除される

### 2.3 申請内容の修正

- **承認前**: [マイページ] → [申請一覧] → 対象の申請を選択 → **[修正]** ボタン
- **承認後**: 修正はできません。一度キャンセルして再申請してください
- **差戻し後**: 差戻し通知に記載の修正指示に従い修正 → **[再申請]** ボタン

## 3. 精算申請の手順

### 3.1 精算申請の作成

1. **[マイページ] → [出張一覧]** から精算対象の出張を選択
2. **[精算申請を作成]** ボタンをクリック（出張申請の承認が完了していないと表示されません）
3. 以下の項目を入力する
   - **実際の交通費**: 利用した交通手段ごとに金額を入力
   - **実際の宿泊費**: ホテル名、宿泊日数、金額を入力
   - **その他経費**: 出張先での会議費等があれば入力
4. 領収書を添付する（次節参照）
5. **[確認画面へ]** → **[精算申請する]** ボタンをクリック

### 3.2 領収書の添付方法

1. 精算申請画面の **[領収書添付]** エリアにファイルをドラッグ＆ドロップ
   - または **[ファイルを選択]** ボタンからアップロード
2. 対応形式: **PDF, JPEG, PNG**（1ファイルあたり **10MB以下**）
3. 各領収書に対して、経費区分（交通費 / 宿泊費 / その他）を選択
4. **スマートフォン撮影のコツ**:
   - 領収書全体が画像に収まるようにする
   - 文字が読める程度の解像度で撮影する
   - 影が入らないようにする
5. 交通系ICカード利用分は、前月利用履歴の CSV エクスポートデータをアップロードできる
   - **[ICカード明細取込]** ボタン → CSV ファイルを選択

### 3.3 精算期限と督促

- 精算期限は出張完了後 **10営業日**
- 期限の3日前にリマインダーメールが自動送信される
- 期限を超過した場合:
  - 超過後も申請は可能だが、**上長への遅延通知**が自動送信される
  - 30営業日を超過すると、**経理部への報告対象**となる

## 4. 承認操作（承認者向け）

### 4.1 承認・差戻し・却下

1. 承認依頼の通知メールのリンクをクリック、または **[マイページ] → [承認待ち一覧]** から対象を選択
2. 申請内容を確認
3. 以下のいずれかを選択:
   - **[承認]**: 申請を承認する
   - **[差戻し]**: 修正を求める（コメント入力必須）
   - **[却下]**: 申請を却下する（理由入力必須）
4. 承認結果は申請者にメールで通知される

### 4.2 代理承認の設定

- **[設定] → [代理承認]** から代理承認者を設定できる
- 代理承認者は同じ部署の **同職位以上** の社員に限る
- 代理期間を設定する（開始日・終了日）
- 代理設定中も本人による承認は可能

### 4.3 一括承認

- 承認待ち一覧でチェックボックスを使って複数選択
- **[一括承認]** ボタンで選択した申請をまとめて承認できる
- ただし、**10万円以上の申請は一括承認の対象外**（個別確認が必要）

## 5. 仮払い申請

1. **[新規申請] → [仮払い申請]** を選択
2. 仮払い対象の出張申請を選択（承認済みの出張のみ表示される）
3. 仮払い希望金額を入力（概算旅費の **80%** が上限）
4. 振込先口座を確認（給与振込口座がデフォルト設定）
5. **[申請する]** ボタンをクリック
6. 承認後、**翌営業日** に振込処理が行われる

## 6. よくあるトラブルと対処法

### 6.1 ログインできない
- ブラウザのキャッシュをクリアして再試行
- Microsoft Entra ID のパスワードが期限切れでないか確認
- それでも解決しない場合は、IT ヘルプデスク（内線: 5000）に連絡

### 6.2 承認者が表示されない
- 人事異動直後は、組織情報の反映に **最大2営業日** かかる場合がある
- 暫定的に、**[承認者を手動選択]** チェックボックスをオンにして、承認者を手動で指定可能

### 6.3 領収書がアップロードできない
- ファイルサイズが10MBを超えていないか確認
- 対応形式（PDF, JPEG, PNG）であることを確認
- ブラウザの拡張機能（広告ブロッカー等）を一時的に無効化して再試行

### 6.4 精算金額が概算と大きく異なる（20%以上の乖離）
- 乖離理由の入力欄が表示されるので、理由を記載する
- 乖離が大きい場合は、承認者による個別確認が入る

### 6.5 過去の申請が表示されない
- デフォルトでは直近 **3か月** の申請が表示される
- **[期間フィルタ]** で表示期間を変更できる
- 過去 **3年分** の申請履歴が保存されている

### 6.6 申請のステータスが「処理中」のまま変わらない
- システムの同期処理は **1日3回**（9:00, 13:00, 17:00）に実行される
- 「処理中」が **2営業日以上** 続く場合は、システム管理者（travelexpense-admin@example.com）に連絡

## 7. スマートフォンでの利用

- TravelExpense はレスポンシブ対応のため、スマートフォンのブラウザからも利用可能
- 専用アプリはありません（2026年4月時点）
- 領収書の撮影→即アップロードが可能
- 一部機能（一括承認、CSVエクスポート等）はPC版のみ
